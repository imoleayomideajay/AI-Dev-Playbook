# Fintech App Evaluation Checklist (Portfolio-Worthy)

Use this checklist to assess whether a fintech analytics demo is credible enough for public portfolio use.

## 1) Business clarity
- [ ] Problem statement is specific and realistic
- [ ] Intended user persona is clear
- [ ] Decision context is explained

## 2) Metric selection
- [ ] Core metrics are defined (e.g., precision, default rate, PAR30)
- [ ] Metric definitions are consistent
- [ ] Trade-offs are acknowledged

## 3) Dashboard usefulness
- [ ] Visuals support real decisions
- [ ] Filters and drill-downs are meaningful
- [ ] KPI cards avoid vanity metrics

## 4) Interpretability and explainability
- [ ] Decision logic is understandable
- [ ] Reason codes or explanations are present
- [ ] Limits of model/rules are documented

## 5) Risk controls
- [ ] Threshold strategy is described
- [ ] Human review path exists
- [ ] High-impact actions have safeguards

## 6) Data assumptions
- [ ] Data schema is documented
- [ ] Label assumptions are stated
- [ ] Synthetic/demo data use is disclosed

## 7) Monitoring readiness
- [ ] Drift indicators are proposed
- [ ] Performance review cadence is defined
- [ ] Alerting/incident ownership is considered

## 8) Visual clarity
- [ ] Layout is readable and professional
- [ ] Charts have clear labels and units
- [ ] Data freshness is visible

## 9) Deployment quality
- [ ] Setup instructions are reproducible
- [ ] Dependencies are pinned reasonably
- [ ] Basic CI/lint checks exist

## 10) README quality
- [ ] Goal, stack, and run steps are clear
- [ ] Scope and limitations are explicit
- [ ] Next improvements are listed

## 11) Recruiter appeal
- [ ] Project communicates business impact
- [ ] Architecture choices are justified
- [ ] Repository is clean and easy to navigate
