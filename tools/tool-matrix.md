# Tool Matrix

Qualitative comparison of popular AI development tools for building real systems.

| Tool | Category | Best for | Skill level | Speed to MVP | Flexibility | Key advantage | Key limitation |
|---|---|---|---|---|---|---|---|
| Codex | AI coding agent | End-to-end implementation assistance across code and repo workflows | Intermediate | High | High | Strong for multi-step coding tasks and repository execution | Requires clear prompting and engineering judgment |
| Cursor | AI-native IDE | Fast code iteration inside a familiar editor workflow | Beginner to Intermediate | High | High | Tight editing + chat workflow for day-to-day development | Context quality depends on workspace hygiene |
| Replit | Cloud dev platform | Quick app prototyping and collaborative building in browser | Beginner to Intermediate | High | Medium | Low setup friction and instant shareability | Less control than fully custom local/cloud stacks |
| Lovable | AI app builder | Rapid UI/app generation from natural language intent | Beginner | High | Medium | Very fast idea-to-interface workflow | Complex customization can require migration/rewrites |
| Streamlit | Python app framework | Data apps, internal tools, analytics dashboards | Beginner to Intermediate | High | Medium | Extremely fast dashboard and data app development | Not ideal for highly custom front-end interactions |
| GitHub Copilot | AI pair programmer | Inline code completion and productivity acceleration | Beginner to Advanced | Medium | High | Seamless in-editor assistance across many languages | Less opinionated workflow guidance for full system design |
| Windsurf | AI development environment | AI-assisted coding with agentic flows and codebase awareness | Intermediate | High | High | Integrates coding and assistant workflow tightly | Team conventions and review discipline still essential |
| Bolt | Prompt-to-app builder | Fast prototyping of full-stack concepts | Beginner to Intermediate | High | Medium | Accelerates from idea to runnable app quickly | Generated architecture may need refactoring for production |
| v0 | UI generation platform | Rapid generation of modern interface components | Beginner to Intermediate | High | Medium | Speeds up UI scaffolding and visual iteration | Backend logic and integration still require engineering effort |

## How to use this matrix

1. Start with your **delivery constraint** (time, skill, deployment target).
2. Choose one **primary build tool** and one **support tool**.
3. Validate with a small benchmark sprint before standardizing.
4. Document why you selected your stack for team consistency.
