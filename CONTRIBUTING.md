# Contributing to AI Dev Playbook

Thanks for your interest in improving this project. We prioritize contributions that make builders faster, clearer, and more successful at shipping real AI systems.

## Ways to contribute

- Improve an existing playbook with clearer execution steps
- Add or update tool comparisons with practical tradeoffs
- Contribute benchmark case studies using the framework
- Propose project blueprints for analytics, fintech, or dashboard workflows
- Improve docs, links, examples, and consistency

## Contribution principles

1. **Be practical.** Prefer implementation guidance over abstract commentary.
2. **Be reproducible.** Include assumptions, stack, and steps.
3. **Be honest about tradeoffs.** Avoid hype and unsupported claims.
4. **Be beginner-aware.** Write so motivated non-experts can execute.

## Getting started

1. Fork the repository
2. Create a branch: `feat/your-change` or `fix/your-change`
3. Make focused changes
4. Validate Markdown rendering and link integrity
5. Open a pull request using the template

## Content standards

- Use concise, GitHub-native Markdown
- Keep sections scannable with headings and bullets
- Add examples where helpful
- Keep terminology consistent across playbooks and tools

## Pull request checklist

- [ ] Change is clear and scoped
- [ ] Docs are internally consistent
- [ ] New files are linked from relevant index pages
- [ ] Claims are practical and non-promotional
- [ ] Contributor has read `CODE_OF_CONDUCT.md`

## Discussion before large changes

For substantial restructuring, open an issue first so we can align on direction.

Thanks for helping make this repo useful to real builders.
